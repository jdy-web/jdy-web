# Pug规范

## 1. 注释规范

{% hint style="info" %}
**目标：所有注释绝不在最终 HTML 中出现，保持交付代码干净、专业、对客户完全不可见。**
{% endhint %}

{% hint style="success" %}
使用 //- 开头，这类注释会被 Pug 完全丢弃，不会出现在生成的 HTML 中
{% endhint %}

```
✅
//- ============================================
//- 页面标题区块 - 2025.12.11 by 张三
//- 业务需求：XXX 项目需求文档 3.2.1 节
//- ============================================
header.header  
    //- 临时隐藏的调试按钮，上线前必须删除  
    //-.debug-btn 调试模式  
    
    h1 #{title}
    
    //- TODO: 2025.01.01 前确认是否需要多语言切换按钮
```

{% hint style="danger" %}
严禁在Pug中直接使用// 且 内联JavaScript（script. 或 script(src)）中禁止任何注释
{% endhint %}

```
⛔
// 会输出到 HTML！绝对禁止！！
p 这是一段文字
// 客户会看到这行注释！！
```

```
⛔
script.
  // 调试用，上线前删除
  console.log('当前用户：', user);

  /* 
    多行注释也会完整显示 
  */
```
